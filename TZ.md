# Техническое задание: Калькулятор углеродного следа

## 1. Цель проекта

Создание веб-сервиса для отслеживания личного углеродного следа. Система помогает эко-активным пользователям визуализировать влияние их повседневных привычек (транспорт, энергия, питание) на окружающую среду через расчёт выбросов CO₂, постановку целей по снижению и наглядные графики.

## 2. Роли пользователей

- **Гость:** просмотр лендинга, информация о проекте, регистрация.
- **Авторизованный пользователь:** добавление записей об активностях, просмотр дашборда с графиками, установка месячной цели по выбросам, просмотр истории своих записей.
- **Администратор:** управление справочником типов активностей и их коэффициентов выбросов через Django Admin.

## 3. Модели данных (сущности)

1. **ActivityType (Тип активности):** справочник видов активностей с коэффициентами выбросов CO₂.
   - _Поля:_ `name`, `category` (transport/energy/food/shopping), `co2_factor` (кг CO₂ на единицу), `unit` (км, кВт·ч, кг).

2. **CarbonLog (Запись пользователя):** конкретное действие пользователя с расчётом выброса.
   - _Поля:_ `user` (ForeignKey), `activity` (ForeignKey на ActivityType), `value` (количество), `date`, `city` (город для погодной корректировки), `notes`.
   - _Вычисляемое поле:_ `co2_emission = value × activity.co2_factor`.

3. **EcoGoal (Цель пользователя):** месячный лимит выбросов для мотивации.
   - _Поля:_ `user` (OneToOneField), `monthly_limit` (кг CO₂), `start_date`, `is_active`.

## 4. Ключевой функционал (User Stories)

- **Добавление активности:** Пользователь выбирает тип активности (например, «Поездка на авто»), вводит значение (25 км), город и дату — система автоматически рассчитывает выброс CO₂ (25 × 0.21 = 5.25 кг) и сохраняет запись.
- **Погодная корректировка:** Для транспортных активностей система запрашивает актуальную погоду через Open-Meteo API. При низкой температуре (ниже 0°C) и осадках коэффициент выбросов увеличивается на 10-25%, что отражает повышенный расход топлива.
- **Дашборд с визуализацией:** Пользователь видит общий след за месяц, прогресс-бар относительно цели, круговую диаграмму распределения выбросов по категориям и линейный график динамики за последние 30 дней (Chart.js).
- **Установка цели:** Пользователь задаёт месячный лимит (например, 150 кг CO₂) — система показывает процент использования лимита и эквивалент в деревьях («ваш след за год поглотят 3 дерева»).
- **История записей:** Пользователь просматривает все свои записи, отсортированные по дате, и может удалять ошибочные записи.

## 5. Внешние интеграции и аналитика

- **Внешние данные:** Коэффициенты выбросов CO₂ взяты из следующих источников:
  - Транспорт и топливо: Приказ Минприроды России от 27.05.2022 №371 ([ссылка](https://www.consultant.ru/document/cons_doc_LAW_423207/))
  - Электроэнергия: Научное исследование "Carbon Footprint of Electricity Produced in the Russian Federation" (MDPI Energies, 2024) ([ссылка](https://www.mdpi.com/1996-1073/18/1/14))
  - Продукты питания: База данных Our World in Data "Environmental impacts of food production" ([ссылка](https://ourworldindata.org/environmental-impacts-of-food))
  - Общественный транспорт: EPA Greenhouse Gas Equivalencies Calculator ([ссылка](https://www.epa.gov/energy/greenhouse-gas-equivalencies-calculator))
  - Товары народного потребления: Исследования жизненного цикла продуктов (Life Cycle Assessment), Ellen MacArthur Foundation

- **Внешний API:** интеграция с **Open-Meteo API** (бесплатный, без API-ключа) для получения актуальных погодных данных. Погодные условия (низкая температура, осадки) влияют на расход топлива транспорта, что корректирует расчёт выбросов CO₂: коэффициент увеличивается на 10-15% при температуре ниже 0°C и на 10% при осадках.

- **Аналитика и визуализация:** использование **Django ORM** (агрегация через `Sum`, `F`) для группировки и расчёта данных по периодам и категориям. Генерация графиков на фронтенде через **Chart.js** (круговая диаграмма категорий, линейный график динамики).
